03 SEO Implementation Details

JSON-LD Structured Data Added

Three structured data blocks are now in the <head>:

  • LocalBusiness — tells Google who Turfli is, where they are, their phone, rating (5.0, 101 reviews), and service area cities.
  • FAQPage — makes Turfli eligible for FAQ rich results in Google Search (expandable Q&A below the listing).
  • Service + ItemList — lists all 9 services with their page URLs for better crawlability and entity association.

Validate your schema: After going live, paste https://turfli.com into search.google.com/test/rich-results to confirm Google can read the structured data correctly.

05 Ads Agent Setup — Google Ads

The ads agent (ads_agent/turfli_ads_agent.py) is a Claude-powered optimization agent. It analyzes campaign performance, recommends changes, and in live mode, applies them. Here's how to connect it to Google Ads.

First-time setup takes about 30 minutes. You'll need access to Google Cloud Console and Google Ads Manager. Complete this once, save the credentials to config.yml, and the agent will work indefinitely.

Step 1: Create a Google Cloud Project

  1. Go to console.cloud.google.com → New Project → Name it "Turfli Ads"
  2. Enable the Google Ads API: APIs & Services → Enable APIs → Search "Google Ads API" → Enable
  3. Create OAuth credentials: APIs & Services → Credentials → Create Credentials → OAuth client ID → Desktop app → Name it "Turfli Ads Agent"
  4. Download the JSON credentials file. You'll get client_id and client_secret from it.

Step 2: Get a Developer Token

  1. Log into Google Ads: ads.google.com
  2. Go to Tools & Settings → API Center
  3. Apply for Basic Access (required for production use). Basic access allows up to 15,000 operations/day — more than enough for Turfli.
  4. Copy your developer token.

Step 3: Generate a Refresh Token

Run this one-time in your terminal from the ads_agent/ directory:

pip install google-ads pyyaml anthropic facebook-business 
# One-time OAuth flow to get refresh token
from google_auth_oauthlib.flow import InstalledAppFlow

flow = InstalledAppFlow.from_client_secrets_file(
    "credentials.json",  # Your downloaded OAuth JSON
    scopes=["https://www.googleapis.com/auth/adwords"]
)
creds = flow.run_local_server(port=0)
print("Refresh token:", creds.refresh_token)

Copy the printed refresh token into your config.yml.

Step 4: Get Your Account IDs

  • customer_id: Your Turfli Google Ads account ID (visible in the top right of ads.google.com, format: XXX-XXX-XXXX, enter without dashes)
  • login_customer_id: Same as customer_id unless you have an MCC (manager account) — then use the MCC ID

Step 5: Update config.yml

Copy config.example.yml to config.yml:

cp config.example.yml config.yml

Fill in all values under the google: section. Set dry_run_by_default: true until you've confirmed the agent's recommendations are sound.

06 Ads Agent Setup — Meta Ads

Step 1: Create a Meta App

  1. Go to developers.facebook.com → My Apps → Create App
  2. Choose "Business" type → Name it "Turfli Ads Agent"
  3. Under "Add Products", add Marketing API
  4. Go to Settings → Basic → copy your App ID and App Secret

Step 2: Get a Long-Lived Access Token

  1. Go to developers.facebook.com/tools/explorer
  2. Select your app → Generate User Token
  3. Add permissions: ads_read, ads_management, business_management
  4. Click Generate Access Token → Copy it
  5. Exchange for long-lived token (60-day expiry) using this URL:
https://graph.facebook.com/v19.0/oauth/access_token?
  grant_type=fb_exchange_token&
  client_id=YOUR_APP_ID&
  client_secret=YOUR_APP_SECRET&
  fb_exchange_token=SHORT_LIVED_TOKEN

Copy the returned access_token to config.yml under meta.access_token.

Step 3: Get Your Ad Account ID

  1. Go to business.facebook.com → Settings → Ad Accounts
  2. Find your Turfli ad account — the ID is the number after act_ (e.g., act_123456789 → your ID is 123456789)
  3. Enter this number (without act_) in config.yml under meta.account_id

Token expiry: Meta long-lived tokens expire after 60 days. You'll need to regenerate them. A future enhancement is to use a System User token via Meta Business Manager which doesn't expire — recommended for production use.

Safety Guardrails Built In

  • Dry run by default — The agent will DESCRIBE what it would do, but not apply any changes, unless you add --live
  • Max budget increase cap — Configured in config.yml as max_budget_increase_pct: 30. The agent will never recommend increasing a budget by more than 30% in a single action.
  • Min budget floor — Configured as min_budget_usd: 10. No campaign will be cut below $10/day.
  • Pause requires explicit reason — Every pause action requires a reason field which is logged in the output.

Setting ANTHROPIC_API_KEY

The agent needs your Anthropic API key. Set it in your terminal session before running:

export ANTHROPIC_API_KEY="sk-ant-your-key-here"

Or add it to config.yml under anthropic_api_key: "sk-ant-..."

Get an Anthropic API key: Sign up at console.anthropic.com. The agent uses claude-opus-4-7. A typical weekly analysis run costs approximately $0.50–$2.00 depending on data volume.

When the Agent Flags Something Critical

The agent will flag campaigns with:

  • CTR below 2% on Google Search — ad copy needs rewriting
  • CPL above $200 on any campaign — budget shift needed
  • Zero conversions in 14+ days — pause recommended
  • Impression share below 30% on Putting Green or Backyard Remodel — budget increase opportunity

Never let the agent run live unreviewed. Always run --action optimize (dry run) first, read the output, and only run --action optimize --live if the recommendations make sense. The agent is highly capable but can't see your calendar, promotions, or business context you haven't told it.
