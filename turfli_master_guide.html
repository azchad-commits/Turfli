03 Phase 2: Deploying the New Site to GoDaddy

One-time file swap — replaces WordPress with the new static HTML site

How this works: turfli.com already points to GoDaddy hosting — that does not change. The WordPress files currently sitting on that server get replaced with the new static HTML files from GitHub. Same domain, same hosting account, same server. The site goes live the moment the new files are uploaded.

Step 1 — Back Up WordPress First

Before touching anything, download a full backup. You cannot undo a file deletion without one.

  1. Log in to GoDaddy → My Products → Web Hosting → Manage
  2. In cPanel → scroll to Files → click File Manager
  3. Navigate into public_html → Select All (Ctrl+A) → Click Compress → Create a .zip archive named wordpress-backup-[date].zip
  4. Right-click the .zip → Download — save it to your computer. This is your safety net.

Do not skip the backup. Once WordPress files are deleted they are gone. The backup takes 5 minutes and protects weeks of work.

Step 2 — Download the New Site from GitHub

  1. Go to github.com → sign in → open the Turfli repository
  2. Click the green Code button → Download ZIP
  3. Extract the ZIP on your computer — you'll get a folder called Turfli-main containing all the site files (index.html, services/, locations/, blog/, etc.)

Step 3 — Delete the WordPress Files from GoDaddy

  1. In cPanel File Manager → navigate to public_html
  2. Select the WordPress files and folders: wp-admin/, wp-content/, wp-includes/, wp-config.php, wp-login.php, wp-cron.php, wp-settings.php, wp-load.php, wp-blog-header.php, wp-trackback.php, wp-comments-post.php, index.php, xmlrpc.php, .htaccess
  3. Delete them all. The backup you created in Step 1 is your safety net.
  4. public_html should now be empty (or contain only unrelated files from other things hosted on the account)

Step 4 — Upload the New HTML Files via File Manager

  1. In cPanel File Manager, make sure you're in public_html
  2. Click Upload in the toolbar → drag in all the files and folders from the Turfli-main folder you extracted in Step 2
  3. Upload everything: index.html, services/, locations/, neighborhoods/, blog/, about-turfli-scottsdale/, contact/, portfolio/, sitemap.xml, robots.txt, all lp-*.html files, and any image files
  4. Wait for the upload to complete, then visit https://turfli.com — the new site should be live immediately.

Alternative: Upload via FTP (Larger Uploads)

For uploading many files at once, FTP is faster than the browser-based File Manager.

  1. In cPanel → scroll to Files → FTP Accounts → note your FTP username, password, and server hostname (usually ftp.turfli.com or the server IP)
  2. Download FileZilla (free at filezilla-project.org) → open it → File → Site Manager → New Site
  3. Enter: Host = your FTP hostname, Protocol = FTP, Logon Type = Normal, User = your FTP user, Password = your FTP password → Connect
  4. In the right pane (remote server), navigate to public_html. In the left pane (local), navigate to the Turfli-main extracted folder.
  5. Select all local files → drag to the right pane → FileZilla uploads everything automatically. Much faster for large batches than the browser uploader.

Clean URLs work automatically: GoDaddy's Apache server serves index.html by default when a folder is requested. So /services/backyard-remodels/ automatically serves services/backyard-remodels/index.html — no configuration needed. All 35+ pages will work correctly at their clean URLs as soon as the files are uploaded.

After Uploading — Immediate Tests

  1. Visit https://turfli.com — confirm the new homepage loads, not the old WordPress site
  2. Click through a service page: https://turfli.com/services/artificial-turf-installation/
  3. Click through a location page: https://turfli.com/locations/scottsdale/
  4. Test on mobile — the site is fully responsive
  5. If you see a WordPress page instead of the new site, clear your browser cache (Ctrl+Shift+R) and try again

Keeping the Site Updated Going Forward

After the initial launch, updating the site is the same process: make changes to the HTML files in GitHub, download the updated ZIP or the specific changed files, upload them to public_html via File Manager or FTP, replacing the old versions. Changes go live immediately on upload — no build process, no server restart.

04 Phase 3: Contact Form Handler

Required — without this, form submissions go nowhere

The contact form in index.html posts to /contact-submit/. Since this is a static HTML site hosted on GoDaddy, there's no server-side code to handle form submissions. You need Formspree — a third-party service that receives the form data and emails it to hello@turfli.com. Setup takes 10 minutes.

Set Up Formspree

  1. Go to formspree.io → Sign up with your email → Dashboard → + New Form
  2. Name it "Turfli Contact Form" → Formspree creates a unique endpoint URL like https://formspree.io/f/xyzabc123
  3. In index.html, find the <form tag in the contact section. Change the action attribute to your Formspree URL:
    <form method="POST" action="https://formspree.io/f/xyzabc123">
  4. In Formspree → your form → Settings → Notifications → enter hello@turfli.com as the email recipient for new submissions
  5. In Formspree → Settings → Redirects → set the success redirect URL to https://turfli.com/thank-you/ — this is the page visitors land on after submitting, and where conversion tracking fires
  6. Upload the updated index.html to GoDaddy (same upload process as Phase 2)
  7. Submit a test form → confirm you receive the email at hello@turfli.com → confirm you land on the thank-you page

Free vs Paid Formspree: Free plan allows 50 submissions/month — enough to start. At $10/month, you get unlimited submissions plus spam filtering. A lead-gen business like Turfli should upgrade once the ads are running and leads are coming in.

Create the Thank-You Page

Create /thank-you/index.html — a simple confirmation page that visitors see after submitting the contact form. This page is where your conversion tracking fires. The page content should confirm the submission and set expectations:

<h1>Got it — we'll be in touch shortly.</h1>
<p>Karl or a member of our team will reach out within 24 hours
to schedule your free on-site design consultation.</p>
<a href="/">Back to Homepage</a>

The full page should match the site design and include both conversion tracking scripts (see Phase 4).

05 Phase 4: Conversion Tracking

Critical — the ads agent cannot optimize without this

This is the most important technical step for ads performance. The ads agent pulls conversion data from Google Ads and Meta to calculate CPL (cost per lead) and optimize budgets. Without conversion tracking properly installed on the website, both platforms only see clicks — not actual leads. Budget decisions based on clicks alone are unreliable and wasteful.

How Conversion Tracking Works

When a visitor fills out the contact form and lands on the /thank-you/ page, tracking scripts fire and tell Google and Meta "this click became a lead." Both ad platforms then use this data to optimize who they show ads to and how much to bid.

Step 1 — Install Google Tag (Global Site Tag)

Get your Google Ads Conversion ID from: Google Ads → Tools & Settings → Conversions → + New conversion action → Website → set up a "Contact form submission" conversion. Google will give you a tag like AW-XXXXXXXXXX.

Add this code to the <head> of every page on the site (index.html, all service pages, all location pages, thank-you page). The easiest way: add it once to a shared template, or add it to each file's <head>:

<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=AW-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'AW-XXXXXXXXXX');
</script>

Replace AW-XXXXXXXXXX with your actual Google Ads tag ID.

Step 2 — Install Google Ads Conversion Event on Thank-You Page

Add this only to the /thank-you/ page, after the global tag above:

<!-- Google Ads Conversion Event -->
<script>
  gtag('event', 'conversion', {
    'send_to': 'AW-XXXXXXXXXX/YOUR_CONVERSION_LABEL',
    'value': 1.0,
    'currency': 'USD'
  });
</script>

Get YOUR_CONVERSION_LABEL from Google Ads → Conversions → select your conversion action → Tag setup → Use Google Tag Manager → copy the label value.

Step 3 — Install Meta Pixel

Get your Pixel ID from: Meta Business Manager → Events Manager → Data Sources → your Pixel → Settings. You'll see a 15-16 digit number.

Add this to the <head> of every page:

<!-- Meta Pixel Code -->
<script>
!function(f,b,e,v,n,t,s)
{if(f.fbq)return;n=f.fbq=function(){n.callMethod?
n.callMethod.apply(n,arguments):n.queue.push(arguments)};
if(!f._fbq)f._fbq=n;n.push=n;n.loaded=!0;n.version='2.0';
n.queue=[];t=b.createElement(e);t.async=!0;
t.src=v;s=b.getElementsByTagName(e)[0];
s.parentNode.insertBefore(t,s)}(window, document,'script',
'https://connect.facebook.net/en_US/fbevents.js');
fbq('init', 'YOUR_PIXEL_ID');
fbq('track', 'PageView');
</script>
<noscript><img height="1" width="1" style="display:none"
src="https://www.facebook.com/tr?id=YOUR_PIXEL_ID&ev=PageView&noscript=1"
/></noscript>
<!-- End Meta Pixel Code -->

Replace YOUR_PIXEL_ID with your actual Pixel ID.

Step 4 — Fire Meta Lead Event on Thank-You Page

Add this only to the /thank-you/ page, after the Meta Pixel base code:

<!-- Meta Lead Conversion Event -->
<script>
  fbq('track', 'Lead');
</script>

Step 5 — Verify Tracking is Working

  1. Install the Meta Pixel Helper Chrome extension → visit turfli.com → confirm it shows the Pixel is firing
  2. Submit a test form → land on the thank-you page → confirm the Pixel Helper shows a "Lead" event fired
  3. In Google Ads → Conversions → your conversion action should show "Recording conversions" (may take up to 24 hours after first test submission)
  4. In Meta Events Manager → Test Events tab → enter your site URL → submit the form → confirm a Lead event appears in real-time

Why this unlocks the full value of the ads agent: Once conversions are tracking, the agent can see CPL (cost per lead) per campaign, not just CPC (cost per click). This allows it to recommend pausing campaigns that are generating expensive clicks with no conversions, and increasing budgets on campaigns that are actually bringing in leads. Without conversion data, every campaign looks equally good.

07 Phase 6: Google Business Profile

Do this the same week the site goes live

Google Business Profile (GBP) is how Turfli shows up in the local map pack — the 3 listings that appear above organic results for searches like "backyard remodel contractor Scottsdale." Currently Turfli is ranked 20+ for that search. The category fix below is the single fastest way to improve that ranking.

  1. Fix the primary business category:
    Log in to business.google.com → Edit Profile → Business category
    Current: "Turf supplier" (wrong — this is why Turfli ranks poorly for backyard remodel)
    Change to: "Landscape Designer" or "Landscaping Service"
    Add secondary categories: "Outdoor Kitchen Contractor", "Patio Contractor", "Artificial Turf Installer"
  2. Update the business description:
    Write 750 characters describing Turfli as a design-build outdoor living contractor. Include: backyard remodels, paver patios, outdoor kitchens, artificial turf, putting greens, pergolas. Mention Scottsdale, Paradise Valley, and luxury neighborhoods. Mention ROC licensed, BBB A+, 5.0 stars.
  3. Add real project photos to GBP:
    After adding photos to the website, upload the same project photos to Google Business Profile → Photos → Add photos. Categorize them as "at work" or "exterior." GBP listings with more photos get more profile views and clicks.
  4. Set up the short review link:
    GBP → Get more reviews → Share review form → Copy the short link. This is your actual review link — update the g.co/kgs/turfli link in index.html (Reviews section) if different.
  5. Enable messaging:
    GBP → Messages → Turn on. Lets potential clients message you directly from the Google Business listing. Response rate and time affect local pack rankings.
  6. Create a GBP post about the new site:
    GBP → Add update → "We just launched our new website with our full portfolio, before-and-after photos, and project gallery. See our work at turfli.com." Include a link and a photo.

After every completed project: Post a photo of the finished project to Google Business Profile with the neighborhood name and what was built (e.g., "Silverleaf backyard — travertine patio, outdoor kitchen, and luxury turf installation"). This is how Turfli builds local pack authority over time.

08 Phase 7: Google Ads Agent Setup

The ads agent (ads_agent/turfli_ads_agent.py) needs 5 credentials from Google. This takes about 30 minutes the first time. Once configured, it runs indefinitely.

Prerequisite: Conversion tracking (Phase 4) must be live and recording at least one conversion before the agent's CPL calculations are meaningful. The agent works fine before that on click and spend data — but wait to act on CPL recommendations until conversions are tracking.

Step 1 — Create a Google Cloud Project

  1. Go to console.cloud.google.com → New Project → Name it "Turfli Ads"
  2. Enable the Google Ads API: APIs & Services → Enable APIs → Search "Google Ads API" → Enable
  3. Create OAuth credentials: APIs & Services → Credentials → Create Credentials → OAuth client ID → Desktop app → Name it "Turfli Ads Agent"
  4. Download the JSON credentials file. Note the client_id and client_secret — you'll need these.

Step 2 — Get a Developer Token

  1. Log into ads.google.com → Tools & Settings → API Center
  2. Apply for Basic Access. Google will review and approve within a few days. Basic access allows 15,000 operations/day — more than enough for Turfli's scale.
  3. Copy your developer token from the API Center page.

Step 3 — Generate a Refresh Token (One-Time)

From the ads_agent/ directory in your terminal:

pip install google-ads google-auth-oauthlib pyyaml anthropic facebook-business

Run this Python script once:

from google_auth_oauthlib.flow import InstalledAppFlow

flow = InstalledAppFlow.from_client_secrets_file(
    "credentials.json",   # Your downloaded OAuth JSON file
    scopes=["https://www.googleapis.com/auth/adwords"]
)
creds = flow.run_local_server(port=0)
print("Refresh token:", creds.refresh_token)

A browser window opens → log in with the Google account that owns the Turfli Google Ads account → authorize access → copy the printed refresh token.

Step 4 — Get Your Customer ID

In Google Ads, your Customer ID is the XXX-XXX-XXXX number in the top right of the interface. Enter it in config.yml without dashes (e.g., 1234567890). If you use a Google Ads Manager Account (MCC), also fill in the login_customer_id with the MCC ID.

Step 5 — Fill in config.yml

cd /Users/chad/Turfli/Turfli/ads_agent cp config.example.yml config.yml

Open config.yml and fill in:

google:
  developer_token: "YOUR_DEVELOPER_TOKEN"
  client_id: "YOUR_OAUTH_CLIENT_ID"
  client_secret: "YOUR_OAUTH_CLIENT_SECRET"
  refresh_token: "YOUR_OAUTH_REFRESH_TOKEN"
  login_customer_id: "1234567890"
  customer_id: "1234567890"
  use_proto_plus: true

Keep dry_run_by_default: true in config.yml for the first month. In dry-run mode the agent describes every change it would make without applying any of them. Review the recommendations for a few weeks before switching to live mode.

  4. Click Generate Access Token → copy the short-lived token

  grant_type=fb_exchange_token&
  client_id=YOUR_APP_ID&
  client_secret=YOUR_APP_SECRET&

The response JSON contains "access_token": "..." — copy this long-lived token.
