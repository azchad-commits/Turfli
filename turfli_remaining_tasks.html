05 Ads Agent Setup

The ads agent code is complete and working. It only needs credentials. Full setup instructions are in Sections 5 and 6 of the original implementation guide. Summary of what's needed:

Google Ads — 4 Credentials Needed

  1. Developer token — From Google Ads → Tools & Settings → API Center. Requires applying for Basic Access.
  2. OAuth client ID + secret — From Google Cloud Console → Create a new project → Enable Google Ads API → Create OAuth credentials.
  3. Refresh token — Run the one-time Python OAuth flow from the original guide (Section 5, Step 3).
  4. Customer ID — Your Turfli Google Ads account number (top-right of ads.google.com, without dashes).

Meta Ads — 3 Credentials Needed

  1. App ID + App Secret — From developers.facebook.com → Create a Business app → Marketing API product → Settings → Basic.
  2. Long-lived access token — From the Graph API Explorer, exchange for 60-day token (see original guide Section 6).
  3. Ad Account ID — From business.facebook.com → Settings → Ad Accounts. Use the number without "act_" prefix.

Running the Agent Once Configured

After filling in ads_agent/config.yml:

cd /Users/chad/Turfli/Turfli/ads_agent pip install -r requirements.txt python turfli_ads_agent.py --action report

Start with a dry run: Keep dry_run_by_default: true in config.yml for the first few weeks. Review the agent's recommendations before applying any changes. The agent describes every action it would take before doing anything.

Meta Token Renewal Reminder

Meta long-lived tokens expire after 60 days. Set a calendar reminder to regenerate at day 55. Future enhancement: switch to a System User token in Meta Business Manager which doesn't expire.

06 Speed Optimization

The guide identified speed as an opportunity. Current hero image is 2048px wide and uncompressed. Target: 70+ PageSpeed on mobile, 90+ on desktop.

Hero Image Optimization

  1. Create three versions of the hero image:
    — fann-hero-desktop.jpg — 1400px wide, 85 quality (target: under 250KB)
    — fann-hero-mobile.jpg — 800px wide, 80 quality (target: under 100KB)
    Use squoosh.app to compress. Use WebP format for even better compression.
  2. Update the hero CSS in index.html to use the optimized versions:
    Replace the current .hero-bg background with the compressed desktop version. Add a mobile media query with the smaller version: 
    .hero-bg{background: linear-gradient(...),
  url('fann-hero-desktop.jpg') center/cover no-repeat}
@media(max-width:768px){
  .hero-bg{background-image: linear-gradient(...),
    url('fann-hero-mobile.jpg');}
}
  3. Update the preload tag in <head> to use the compressed filename:
    <link rel="preload" as="image" href="fann-hero-desktop.jpg">
  4. Compress all other images — Portfolio photos, before/after photos, and Instagram grid images should all be under 200KB each. Run them through squoosh.app before uploading.

Additional Speed Wins

  • Consider self-hosting the Google Fonts instead of loading from fonts.googleapis.com — saves one render-blocking request
  • All <img> tags already have loading="lazy" and explicit width/height attributes — good
  • After going live, run a PageSpeed Insights test and address any red flags
